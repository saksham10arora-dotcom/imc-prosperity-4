# 02 — INTARIAN_PEPPER_ROOT: The Asymmetry Signal

## The Question

INTARIAN_PEPPER_ROOT was an 80-unit position limit asset that trended upward on all 3 training days. Before we could trade it, we needed to understand: what drives the price, and can we predict direction?

## Statistical Profile

| Statistic | Value | Interpretation |
|---|---|---|
| Drift | **+0.108 per tick** | ~1,000 points/day upward |
| Noise std | **~1.7** | |
| Start price | ~9,998 | |
| End price (day 0) | ~11,100 | After 10,000 ticks |
| Spread | ~13 ticks | Scales with price level |

**All 3 training days trended up.** No crash in Round 1 data.

## Bot Decomposition

Three bots maintained the PEPPER order book:

### Bot 1 — Inner Ladder Maker
- **Presence:** Always present
- **Offset:** ±6.5 from wall_mid
- **Volume:** Uniform [8, 10]

### Bot 2 — Outer Depth Bot
- **Presence:** ~65% of ticks
- **Offset:** ±9.5 from wall_mid
- **Volume:** Uniform [17, 24]

### Bot 3 — Taker
- **Presence:** ~3% per tick
- **Volume:** Uniform [3, 7]
- **Direction:** Conditioned on book asymmetry — fires at 5× the normal rate at extreme asymmetry values, direction matching the signal.

## The Discovery: Level Asymmetry Signal

We computed `asymmetry = n_bid_levels - n_ask_levels` at every tick and measured the conditional probability of price going UP next tick.

| Asymmetry | P(price UP) | Interpretation |
|---|---|---|
| ≤ −1 (more ask levels) | **91%** | Strong buy signal |
| 0 (balanced) | **82%** | Neutral-bullish |
| ≥ +1 (more bid levels) | **49%** | Coin flip — trend pausing |
| Base rate | **76%** | Uptrend baseline |

### Statistical Validation

| Test | Value |
|---|---|
| Chi-squared p-value | **< 0.001** |
| Cramér's V | **0.306** (after Bonferroni correction: 0.274) |
| Consistency | Holds across all 3 days independently |

**The signal is real.** More ask levels than bid levels predicts UP with 91% accuracy.

### Why It Couldn't Be Monetized

1. **Position saturation:** When you're already long 80 (max), knowing "91% up" doesn't help — you can't go more long.
2. **The one actionable use:** Reduce position when asym ≥ +1 (since trend is pausing). We tried this in V16 — it scored **lower** live (10.4K vs 10.7K).
3. **Trend indicator already captures it:** The regression slope + majority vote already tracks the same directional information.

**Lesson learned:** A statistically significant signal ≠ a profitable signal. Edge requires the signal to add information BEYOND what's already being exploited.

## The Skew/Cancel Bot Connection

The asymmetry signal was mechanically generated by a skew/cancel bot that created one-sided book states (1×2 or 2×1 configurations). This bot's behavior was consistent — it wasn't informed, but its presence/absence created a reliable directional fingerprint.
